# CPython core team over time — static site

Runs the same Python computation as the desktop `core_devs_activity.py`
inside the browser, via Pyodide (CPython + matplotlib compiled to WASM).

## Files

```
pyodide_web/
  index.html      — page; the Python program is embedded in <script type="text/python">
  style.css
  data.js         — auto-generated by `core_devs_activity.py --only export-web`
  README.md       — this file
```

## How it works

1. Browser loads `pyodide.js` from jsDelivr (~50 KB JS, then ~10 MB WASM core).
2. `pyodide.loadPackage(["matplotlib"])` downloads matplotlib + numpy (~15 MB).
3. The inline `<script type="text/python">` block is read via `textContent`
   (the browser ignores `type="text/python"` so it never tries to execute it
   as JavaScript).
4. `pyodide.globals.set(...)` hands the data and UI settings into the Python
   global namespace; `pyodide.runPythonAsync(code)` executes the program.
5. matplotlib's figure is saved as an in-memory PNG, base64-encoded and
   dropped into an `<img>`. This sidesteps the Pyodide canvas-backend
   placement story; the chart shows up wherever the `<img>` tag is.

## Performance characteristics

| Stage | Time |
| ---- | ---- |
| First load (cold cache) | 5–20 s (mostly WASM download) |
| Subsequent loads | 1–2 s (Pyodide bootstrap, packages in browser cache) |
| Per-render | 50–200 ms (Python execution + matplotlib + base64) |

## Local viewing

Works from `file://`: open `index.html` directly. The data lives in `data.js`
which is loaded via a plain `<script>` tag, not blocked by the same-origin
policy. Pyodide and matplotlib are pulled from jsDelivr over HTTPS regardless.

## Deploy to GitHub Pages

Commit this directory to a repo, then in **Settings → Pages** set the source
to the `main` branch and folder to `/pyodide_web` (or rename to `/docs` —
the more conventional path).

## Refresh the data

```sh
GITHUB_TOKEN=$(gh auth token) python3 ../core_devs_activity.py \
    --refresh-roster --refresh-activity
python3 ../core_devs_activity.py --only export-web
git add data.js && git commit -m "Refresh roster + activity"
```
